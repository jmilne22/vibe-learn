// vibe-learn

Self-hosted
course platform

Markdown lessons, YAML exercises, one build command. The engine handles rendering, spaced repetition, daily practice, analytics, flashcards, and offline support. You just write the content. No backend, no database, no accounts — static files you can host anywhere.

>_
Markdown lessons
Syntax-highlighted code blocks, side-by-side comparisons, callout boxes. Write content, not HTML.
[ ]
Exercises & challenges
Warmups and challenges with multiple variants, difficulty modes (easy→hard), progressive hints, annotated solutions, and self-rating.
SRS
Spaced repetition
SM-2 algorithm schedules reviews for exercises and flashcards. Items you struggle with come back sooner.
dp
Daily practice
Swipe-style sessions that auto-select what to practice: review due items, drill weak spots, or discover new material.
FC
Flashcards
Flip-and-rate cards with keyboard shortcuts. Filter by module, review mode (random, due, weakest), and configurable deck size.
25m
Pomodoro timer
Built-in session timer with presets (25/5, 50/10, 90/20). Focus mode hides distractions. Thinking timer locks hints while you work.
|||
Analytics
Weak concept reports, module strength rankings, activity heatmap, streak tracking, and rating breakdowns.
RW
Real-world challenges
Open-ended engineering problems sourced from real company repos (Cloudflare, Google, Uber, Fly.io, and more). Requirements, hints, acceptance criteria — no solutions.
AL
Algorithm practice
Session-based algorithm drills across 10 categories with progressive difficulty variants. Discover, review, or drill your weakest problems.
{}
Plugin system
Features are self-contained plugins with a manifest.json. Add flashcards, challenges, or analytics to any course by dropping in content files.
css
10 themes
Gruvbox, Solarized, Everforest, OLED dark, Persona 5, Factorio, and more. Switch from the sidebar, persists across sessions.
SW
Offline & portable
Service worker caches everything for offline use. All progress in localStorage — export, import, or nuke. Host on GitHub Pages, a USB stick, anywhere.
// zero config

Everything is auto-discovered. No registries, no config files to update, no wiring. Drop files in the right place and the build figures out the rest.

>
Drop a course folder → it appears on the site
The build scans courses/ for any directory with a course.json. Add a folder, rebuild, done. Each course gets isolated progress via its own localStorage prefix.
courses/python/ + course.json → auto-discovered
>
Drop a plugin folder → it wires into every course
Plugins live in engine/plugins/ with a manifest.json. The sidebar, dashboard, data backup, and navigation are all driven by discovered plugins. No hardcoded feature lists.
engine/plugins/my-feature/ + manifest.json → auto-wired
>
Add a content file → the feature activates for that course
Each plugin declares a content pattern. If a course has matching files, it gets the feature. No files? The page doesn't exist — no dead links, no errors. The Go course has algorithm problems; Rust doesn't. Both just work.
content/algorithms/algorithms.yaml exists → algorithms page active
// how it works
01
Write content
Lessons in markdown, exercises in YAML. Drop them in a course directory.
courses/go/content/ lessons/*.md exercises/*.yaml flashcards/*.yaml
02
Build
One command. Markdown is rendered, YAML compiled to JS, everything bundled.
npm run build
03
Deploy anywhere
Pure static files. GitHub Pages, Netlify, S3, a USB stick. No server needed.
dist/ index.html go/ rust/ ...
Fork on GitHub
Replace the content, build, deploy. The engine is generic — the courses are just data.
